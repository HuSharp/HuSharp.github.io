### 1.最长子序列  LCS 77

【题目】

给出两个字符串，找到最长公共子序列(LCS)，返回LCS的长度。

```
样例 1:
	输入:  "ABCD" and "EDCA"
	输出:  1
	
	解释:
	LCS 是 'A' 或  'D' 或 'C'


样例 2:
	输入: "ABCD" and "EACB"
	输出:  2
	
	解释: 
	LCS 是 "AC"
```

最长公共子序列的定义：

- 最长公共子序列问题是在一组序列（通常2个）中找到最长公共子序列（注意：不同于子串，LCS不需要是连续的子串）。该问题是典型的计算机科学问题，是文件差异比较程序的基础，在生物信息学中也有所应用。

【解析】

**1、确定状态**

 设A长度是m，B长度是n。现在我们考虑最优策略产生出的最长公共子串（虽然还不知道是什么）最后一步：观察A[m-1]和B[n-1]这两个字符是否作为一个对子在最优策略中

最长公共子串也是公共子串：**长度是L-->>选定了L个对应的对子**

主要分为以下几种情况：

情况一：对子中没有 A[m-1]

![image-20200925131843974](/assets/blog_image/2020-09-20-DP-include3/image-20200925131843974.png)

推论：A和B的最长公共子串就是 A前 m-1 个字符和B前 n 个字符的最长公共子串

情况二：对子中没有 B[n-1]

![image-20200925132551504](/assets/blog_image/2020-09-20-DP-include3/image-20200925132551504.png)

 推论：A和B的最长公共子串就是 A前 m 个字符和B前 n-1个字符的最长公共子串

情况三：对子中有A[m-1]-B[n-1]

![image-20200925132821366](/assets/blog_image/2020-09-20-DP-include3/image-20200925132821366.png)

推论：A和B的最长公共子串就是A前 m-1 个字符和B前 n-1 个字符的最长公共子串 + A[m-1]

子问题：

- 要求A[0...m-1]和B[0...n-2] 的最长公共子串，A[0..m-2]和B[0...n-1]的最长公共子串和A[0..m-2]和B[0..n-2]的最长公共子串
- 原来是求A[0..m-1]和B[0..n-1]的最长公共子串子问题

因此构成子问题

状态：设f[i] [j] 为 A 前 i 个字符 A[0..i-1] 和 B前 j 个字符 B[0.j-1] 的最长公共子串的长度。

**2、状态方程**

![image-20200925133242746](/assets/blog_image/2020-09-20-DP-include3/image-20200925133242746.png)

**3、初始条件和边界情况**

初始条件：空串和任何串的最长公共子串长度是0.

- f[0] [j] = 0，j = 0...n
- f[i] [0] =0， i = 0...m

**4、计算顺序**

由于从转移方程可知，当前状态只与本行和上一行有关，因此直接采用行列式，且可以进行滚动数组优化。

最后答案为 f[m] [n]

时间空间复杂度均为 O(MN）

### 3.交叉字符串 29

【题目】

给出三个字符串:*s1*、*s2*、*s3*，判断*s3*是否由*s1*和*s2*交叉构成。

**样例 1：**

```
输入:
"aabcc"
"dbbca"
"aadbbcbcac"
输出:
true
```

**样例 2：**

```
输入:
""
""
"1"
输出:
false
```

**样例 3：**

```
输入:
"aabcc"
"dbbca"
"aadbbbaccc"
输出:
false
```

挑战

要求时间复杂度为O(n2)或者更好

【解析】

1、确定状态

- 首先，如果X的长度不等于A的长度+B的长度，直接输出False

- 设A长度是m，B长度是n，X的长度是m+n

- 最后一步：假设X是由 A 和 B 交错形成的，那么 X 的最后一个字符 X[m+n-1]

  - 要么是A[m-1]

    那么X[0..m+n-2]是由A[0..m-2]和B[0.n-1]交错形成的

  - 要么是B[n-1]
    那么X[0..m+n-2]是由A[0..m-1]和B[0..n-2]交错形成的

- 因此构成子问题：

- 题目求需要知道 X[m+n-1] 是否由 A[0..m-1] 和 B[0..n-1] 构成

- 现在需要知道 X[0..m+n-2] 是否由A[0..m-2]和B[0..n-1] 形成，以及 X[0..m+n-2] 是否由A[0..m-1]和B[0..n-2]交错形成。

设状态：设f[s] [i] [j] 为 X 前 s 个字符是否由 A 前i个字符A[0.i-1]和B前j个字符 B[0..j-1] 交错构成。

可以化简一下：由于 s = i + j， 因此，直接f[i] [j] 表示 X 的前 i+j 个字符是否由 A 前i个字符A[0.i-1]和B前j个字符 B[0..j-1] 交错构成。

 2、转移方程

![image-20200928204258434](/assets/blog_image/2020-09-20-DP-include3/image-20200928204258434.png)

3、初始条件（序列型考虑空

f[0] [0] = True 表示空串是由 A 的空串和 B 的空串组成

如果 i==0 那么不考虑情况 1 ，如果 j==2 那么不考虑情况2

 4、计算顺序

行列式，自上而下

答案为 f[m] [n]

时间复杂度：O(MN)  空间复杂度：O(MN)

可以通过滚动数组优化





### 4.编辑距离 119

给出两个单词word1和word2，计算出将word1 转换为word2的**最少操作次数**。

你总共三种操作方法：

- 插入一个字符
- 删除一个字符
- 替换一个字符

"horse"
"ros"

解释: 
horse -> rorse (替换 'h' 为 'r')
rorse -> rose (删除 'r')
rose -> ros (删除 'e')
```

```

"execution"
输出: 5

- ![image-20201001110811690](/assets/blog_image/2020-09-20-DP-include3/image-20201001110811690.png)
- 情况三
- ![image-20201001110855017](/assets/blog_image/2020-09-20-DP-include3/image-20201001110855017.png)
- **应当记住情况四！！！！最简单情况：相等情况**
- ![image-20201001110913718](/assets/blog_image/2020-09-20-DP-include3/image-20201001110913718.png)

子问题：

- 之前：求A[0..m-1]和B[0..n-1]的最小编辑距离
- 现在：要求A[0..m-1]和B[0..n-2]的最小编辑距离，A[0..m-2]和B[0..n-1]的最小编辑距离和A[0..m-2]和B[0..n-2]的最小编辑距离子问题

- 状态：设 f[i] [j] 为A前i个字符A[0.i-1]和B前j个字符B[0.j-1]的最小编辑距离

行列式从左至右 从上至下。

### 5、不同子序列 118

给定字符串 `S` 和 `T`, 计算 `S` 的所有子序列中有多少个 `T`.

子序列字符串是原始字符串删除一些(或零个)字符之后得到的字符串, 并且要求剩下的字符的相对位置不能改变. (比如 `"ACE"` 是 `ABCDE` 的一个子序列, 而 `"AEC"` 不是)

**样例 1:**

```
输入: S = "rabbbit", T = "rabbit"
输出: 3
解释: 你可以删除 S 中的任意一个 'b', 所以一共有 3 种方式得到 T.
```

**样例 2:**

```
输入: S = "abcd", T = ""
输出: 1
解释: 只有删除 S 中的所有字符这一种方式得到 T
```

挑战时间复杂度 O(n2n^2n2), 空间复杂度 O(n) 的算法.

如果你不知道如何优化空间, O(n2n^2n2) 的空间复杂度也是可以通过的.

【解析】

1、确定状态

采用对子方式来思考——与最长公共子序列不同的是，B的每一个字符都必须出现在一个对子里。所以这题的“最后一步”以B为出发点。

最后一步即为 B[n-1] 和 A[m-1]，是否结成对子

- 若结成对子：之前便需要 求B[0..n-2]在A[0..m-2]中出现多少次
- 若未结成对子：之前便需要 求B[0..n-1]在A[0..m-2]中出现多少次

因此构成子问题
状态：设 f[i] [j] 为B前j个字符B[0...j-1]在A前i个字符A[0..i-1]中出现多少次

2、转移方程

![image-20201001123722373](/assets/blog_image/2020-09-20-DP-include3/image-20201001123722373.png)

直接相加就可，不遗漏，也不冲突

3、初始条件

现思考 f[i] [0] 应该初始化为 多少？

首先 f[i] [0] 只在 f[i+1] [1] 中被调用

f[i+1] [1] 调用 f[i] [0] 和 f[i] [1]， 调用 f[i] [0] 的情况为：此时 A[i] 和 B[1] 结成对子，应当算作一种情况后 再与 f[i] [1] 方案数相加。 

- 如果B是空串，B在A中出现次数是1
- f[i] [0] = 1（ i=0，1，2，..…，m）
- 如果A是空串而B不是空串，B在A中出现次数是 0
  -f[0] [j] = 0 ( j=1，2.…，n）
- f[0] [0] 也为 1



4、计算顺序

行列式 先初始化 f[0] [0] ....f[0] [n]

再按行按列算，最终返回 f[m] [n]

时间复杂度：O(MN)  空间复杂度：O(MN)





### 6、正则表达式匹配 154

实现支持**'.'**和**'\*'**的正则表达式匹配。

'.'匹配任意一个字母。

'*'匹配零个或者多个前面的元素。

匹配应该覆盖整个输入字符串，而不仅仅是一部分。

需要实现的函数是：bool isMatch(string s, string p)

isMatch("aa","a") → false

isMatch("aa","aa") → true

isMatch("aaa","aa") → false

isMatch("aa", "a*") → true

isMatch("aa", ".*") → true

isMatch("ab", ".*") → true

isMatch("aab", "c*a*b") →  true

**样例 1:**

```
输入："aa"，"a"
输出：false
解释：
无法匹配
```

**样例 2:**

```
输入："aa"，"a*"
输出：true
解释：
'*' 可以重复 a
```

【解析】

1、确定状态

属于：双序列型动态规划
设A长度是m，B长度是n，现在我们考虑A和B如何匹配

**最后一步**：关注最后的字符
主要取决于正则表达式B中最后的字符B[n-1]是什么。

- 如果B[n-1]是一个正常字符（非 . 非 * ），则如果A[m-1]=B[n-1]，能否匹配取决于A[0..m-2] 和 B[0..n-2] 是否匹配；**否则不能匹配**
- 如果B[n-1]是 ' . '，则A[m-1]一定是和 ' . ’ 匹配，之后能否匹配取决于A[0..m-2]
  和B[0..n-2]是否匹配
- 如果B[n-1]是’ * ' ，它代表 B[n-2]=c 可以重复0次或多次，它们是一个整体 c * ，需要考虑A[m-1]是0个c，还是多个c中的最后一个。
  - A[m-1]是0个c，能否匹配取决于A[0..m-1]和B[0..n-3]是否匹配----> 去掉 ' c* ' 相当于 没有  c *  。
  - A[m-1]是多个c中的最后一个，能否匹配取决于A[0..m-2] 和 B[0.n-1]是否匹配
    这种情况必须 A[m-1] = c 或者 c = ' . ’

**子问题**

- 原要求 A 前 m 个字符 和 B 前 n 个字符能否匹配，
- 现在需要知道 A 前 m-1 个字符和 B 前 n-1 个字符（ 非字符匹配成功 或 .  ），A 前 m-1 个字符和 B 前 n 个字符 （ c * 相当于多个 c 的一个）以及 A 前 m 个字符和 B 前 n-2 个字符能否匹配（去掉 c * ）
- 因此子问题
- 设状态：设 f[i] [j] 为 A 前 i 个字符 A[0.i-1] 和 B 前 j 个字符 B[0..j-1] 能否匹配

2、转移方程

![image-20201002130756669](/assets/blog_image/2020-09-20-DP-include3/image-20201002130756669.png)

3、初始条件和边界情况

空串和空正则表达式匹配：f [0] [0] = TRUE
空的正则表达式不能匹配 长度 > 0 的串
-f [1] [0]=....=f [m] [0]=FALSE

因为空串可能可以和匹配式匹配，因此需要进行 DP 来计算

**注意**：f[0] [1..n] 也用动态规划计算，但是因为没有A[-1]，所以只能用第二种情况的 f[i] [j-2]

4、计算顺序

与之前一样 时间复杂度（计算步数）O（MN），空间复杂度（数组大小）O（MN）

空间可优化为 O(N)

【题目】

​	在计算机世界中, 由于资源限制, 我们一直想要追求的是产生最大的利益.	
​	现在，假设你分别是 m个 `0` 和 n个 `1` 的统治者. 另一方面, 有一个只包含 `0` 和 `1` 的字符串构成的数组.
​	现在你的任务是找到可以由 m个 `0` 和 n个 `1` 构成的字符串的最大个数. 每一个 `0` 和 `1` 均只能使用一次

**样例1**

```
输入：
["10", "0001", "111001", "1", "0"]
5
3
输出： 4
解释：这里总共有 4 个字符串可以用 5个 0 和 3个 1 来构成, 它们是 "10", "0001", "1", "0"。
```

**样例2**

```
输入：
["10", "0001", "111001", "1", "0"]
7
7
输出： 5
解释： 所有字符串都可以由7个 0 和 7个 1 来构成.
```

**注意事项**

1. 给出的 `0` 和 `1` 的个数不会超过 `100`
2. 给出的字符串数组的大小不会超过 `600`

【解析】

1、确定状态

最后一步：最优策略组成了最多的01串，其中有没有最后一个字符串S T-1

- 情况1：没有ST-1
  需要知道前T-1个01串中，用m个0和n个1最多能组成多少个01串
- 情况2：有ST-1
  设 第 T-1 个01串中有aT-1 个0，b T-1个1
  需要知道前 T-1 个01串中，用 m-aT-1 个 0 和 n-bT-1 个 1 最多能组成多少个01串

子问题：

原本是要求前 T 个 01 串，用 m 个 0 和 n 个 1 最多能组成多少个 01 串

现在要求前T-1个01串中，用m个0和n个1最多能组成多少个01串 或 前 T-1 个01串中，用 m-aT-1 个 0 和 n-bT-1 个 1 最多能组成多少个01串

因此构成子问题。

值得注意的是：0 和 1 的个数一直在变化，如何记录？

- 解决方案：直接放入到状态中

状态：设 f[i] [j] [k]  为前 i 个 01 串最多被有多少个被 j 个 0 和 k 个 1 组成

### 2、最小调整代价 91

给一个整数数组，调整每个数的大小，使得相邻的两个数的差不大于一个给定的整数target，调整每个数的代价为调整前后的差的绝对值，**求调整代价之和最小是多少**。

样例

```
样例 1:
	输入:  [1,4,2,3], target=1
  输出:  2 【2，3,2,3】

样例 2:
	输入:  [3,5,4,7], target=2
	输出:  1
	
```

注意事项

你可以假设数组中每个整数都是**正整数**，且小于等于**100**。（1-100）

【解析】

看到 最 字，想到最值型DP。

首先可以证明，最优策略中B的每个元素也一定是不超过100。由于原数组是处于 1-100 ，因此将 B 中每个小于 1 的数改成1，大于 100 的数改成 100，会减少相邻 差值。那么总的修改代价更小，且仍然满足B的任意两个相邻元素的 target。

比如 原所谓最优策略得到数组为 ：-3 2 105 104 8------>改为 1 2 100 100 8

-3 与 2 之间的差值变小，105 与 104 差值也变为 0 （差值最小就为 0 ），105 与 8 也自然变小。因此必然会使修改代价更小。

1、确定状态

**最后一步：**

将A改成B，A[n-1] 改成 X ，这一步代价是 |A[n-1] - X| ， 需要确保|X-B[n-2]l<=Target。

**子问题：**

原本需要求 前面 n 个元素 A[0..n-1] 改成 B[0..n-1] 的最小代价之和

现在需要知道前面 n-1 个元素 A[0..n-2] 改成 B[0..n-2] 的最小代价之和，且B[0..n-2]中任意两个相邻的元素的差不超过Target。

因此构成子问题

但是现在有一个问题，改 A[n-1] 时不知道 B[n-2] 是多少。------>>>解决方案：加上 B[n-2] 作为一个状态。

设 f[i] [j] 表示将 A 前 i 个元素改成 B 的最小代价，确保前 i 个改好的元秦中任意两个相邻的元素的差不超过 Target，并且 A[i-1] 改成 j

如果 A[i-1] 改成j，A[i-2] 就必须改成 j-Target <= k <=  j+Target

初始条件：A的第一个元秦可以变换成任意数字

- 因为之前没有相邻的元素
- f[1] [0] = | j-a[0] |  ( j=1，2，…，100）

### 3、K数之和

给定 *n* 个不同的正整数，整数 *k*（*k* <= *n*）以及一个目标数字 *target*。　
在这 *n* 个数里面找出 *k* 个数，使得这 *k* 个数的和等于目标数字，求问有多少种方案？

**样例1**

```plain
输入:
List = [1,2,3,4]
k = 2
target = 5
输出: 2
说明: 1 + 4 = 2 + 3 = 5
```

**样例2**

```plain
输入:
List = [1,2,3,4,5]
k = 3
target = 6
输出: 1
说明: 只有这一种方案。 1 + 2 + 3 = 6
```

【解析】

背包问题

数组A：各个物品的重量
Target：背包最大称重
使得和是Target：背包正好装满

难点在于必须为 K 个

1、确定状态

最后一步：最后一个数An-1是否选入这K个数

- 情况一（An-1不选入）：需要在前 n-1 个数中选 K 个数，使得它们的和是Target
- 情况二（An-1选入）：需要在前 n-1 个数中选 K-1 个数，使得它们的和是 Target-An-1
- 很明显构成子问题

因此需要知道还有几个数可选，以及它们的和需要是多少：采用**序列加状态**

状态：f[i] [k] [s] 表示有多少种方法可以在前 i 个数中选出 k 个，使得它们的和是 s

2、转移方程

![image-20201005104815811](/assets/blog_image/2020-09-20-DP-include3/image-20201005104815811.png)

3、初始条件和边界情况

背包问题初始考虑：

0个物品可以拼出重量0，最大总价值是0
0 个物品不能拼出大于0的重量

因此初始：

```
f[0][0][0]=1
f[0][0][s]=0, s=1,2,.…,Target
```

如果s<Ai-1，只考虑情况一 f[i-1] [k] [s]

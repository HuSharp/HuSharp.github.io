### 1、可行性背包问题

#### 1.背包问题 92

在n个物品中挑选若干物品装入背包，最多能装多满？假设背包的大小为m，每个物品的大小为A[i]。即询问是否存在。

**样例**

```
样例 1:
	输入:  [3,4,8,5], backpack size=10
	输出:  9

样例 2:
	输入:  [2,3,5,7], backpack size=12
	输出:  12
	
```

**挑战**

O(n x m) 的时间复杂度 and O(m) 空间复杂度
 如果不知道如何优化空间O(n x m) 的空间复杂度也可以通过.

**注意事项**

你不可以将物品进行切割。

**【解答】**

背包问题中，数组的大小和总承重有关。

**1、确定状态**：又分为 最后一步 和 化为子问题 

  1. **最后一步**：最后一个物品(重量 An-1)是否进入背包

     分为两种情况：

     - 情况一：如果前N-1个物品能拼出W，当然前N个物品也能拼出W
     - 情况二：如果前N-1个物品能拼出W-An-1，再加上最后的物品An-1，拼出W

  2. **化为子问题：**

     - 要求前N个物品能不能拼出重量0，1.…，M

     - 需要知道前N-1个物品能不能拼出重量0，1…，M
       因此化为子问题

     - 状态：设f[w]=能否用前 i 个物品拼出重量w（TRUE/FALSE）

       ![image-20200904231806339](assets/blog_image/2020-09-02-DP-include2/image-20200904231806339.png)

**2、转移方程**：

![image-20200904231828517]( assets/blog_image/2020-09-02-DP-include2/image-20200904231828517.png)

**3、初始条件以及边界情况**

- f[0] [0] = TRUE：0个物品可以拼出重量0

- f[0] [1..M] = FALSE：0个物品不能拼出大于0的重量

  

  边界情况：
  f[i-1] [w-Ai] 只能在w >= Ai时使用

![image-20200904232111316](assets/blog_image/2020-09-02-DP-include2/image-20200904232111316.png)

**4、计算顺序**

先初始化 f[0] [0] ... f[0] [M]，再算下面的行，纵坐标代表前几个物品。

时间复杂度 ： O(MN),  空间复杂度 :O(数组大小)，可优化为一个数组，因为只取决于前面的几个值。



### 2、计数型背包问题

#### 1.背包问题V 563

给出 n 个物品, 以及一个数组, `nums[i]` 代表第i个物品的大小, 保证大小均为正数, 正整数 `target` 表示背包的大小, 找到能填满背包的方案数。 **`每一个物品只能使用一次`**

**样例**

给出候选物品集合 `[1,2,3,3,7]` 以及 target `7`

```
结果的集合为:
[7]
[1,3,3]

返回 2
```

【解答】

​	和前一题Backpack不一样的是，我们需要求出有多少组合的和是 Target 而不是能不能拼出 Target 。当然，如果能知道这N个物品有多少种方式拼出0，有多少种方式拼出1，.，有多少种方式拼出Target，也就得到了答案。

**1、确定状态**：又分为 最后一步 和 化为子问题

     - 现在是需要 **情况一的个数+情况二的个数=用前N个物品拼出W的方式**

  2. **化为子问题：**

     - 需要知道前N-1个物品有多少种方式拼出重量0，1…，M

#### 2.背包问题 IV 564

给出一个都是正整数的数组 `nums`，其中没有重复的数。从中找出所有的和为 `target` 的组合个数。

**样例1**

```
输入: nums = [1, 2, 4] 和 target = 4
输出: 6
解释:
可能的所有组合有：
[1, 1, 1, 1]
[1, 1, 2]
[1, 2, 1]
[2, 1, 1]
[2, 2]
[4]
```

**样例2**

```
输入: nums = [1, 2] 和 target = 4
输出: 5
解释:
可能的所有组合有：
[1, 1, 1, 1]
[1, 1, 2]
[1, 2, 1]
[2, 1, 1]
[2, 2]
```

**注意事项**

一个数可以在组合中出现多次。

【分析】

与 背包问题V 的区别：组合中数字可以按不同的顺序，比如1+1+2与1+2+1算两种组合。不能先处理第一个物品，再处理第二个物品，…

看上去更难，其实更简单——即考虑最后一个物品重量是 A0 - AN-1时，各自有多少种组合，并将各个组合数加起来。

**1、确定状态**：又分为 最后一步 和 化为子问题 

    1. **最后一步**：最后一个物品重量是啥，是否进入
       - 如果最后一个物品重量是A0，则要求有多少种组合能拼成 Target-A0 
       - 如果最后一个物品重量是A1，则要求有多少种组合能拼成Target-A1
       - ....
       - 如果最后一个物品重量是AN-1，则要求有多少种组合能拼成Target-AN-1
    2. **化为子问题：**
    
       - 原问题：要求前N个物品有多少种方式拼出重量 target
       - 子问题：需要知道前N-1个物品有多少种方式拼出重量 target - A[k]
         因此化为子问题
       - 状态：设f[i] = 有多少种组合能拼出重量i

**2、转移方程**

![image-20200905082612711](/assets/blog_image/2020-09-02-DP-include2/image-20200905082612711.png)

**3、初始化和边界情况**

- f[0]= 1 ——>> 有1种组合能拼出重量0（什么都不选）
- 如果 i<Aj，则对应的 f[i-Aj] 不加人 f[i]
  A0=1，A1=2，A2=4
  f[3]=f[3-A0]+f3-A1]

**4、计算顺序**

```
f[i] = f[i-A0] + f[i-A1]+.…+f[i-An-1]
f[0]=1
计算 f[1]，f2]…，f[Target]
结果为f[Target]
```



#### 背包问题V 变式：

输出可行的某种方案数

由上面分析可知，现在 f[i] 表示有多少种构成方式。

1.pai 数组的增加讨论

现在增加一个 pai 一维数组，pai[i] 表示：如果构成重量 i 的方案数 >=1 ，那么 pai[i] 就表示构成该重量的最后一个数字（任意一种方案的）

 那么 现在重量为 target ，因此想要输出结果，便需要得到 pai[target] ，继续需要得到 pai[target - pai[target]].....

2.pai 数组怎么得到？

按照递归方程， `f[i] += f[i - nums[j]];`，如果 `f[i - nums[j]] > 0 ` 那么：`pai[i] = nums[j];`。

```
    public int backPackVI_output(int[] nums, int target) {
        int[] f = new int[target + 1];
        f[0] = 1;
        int[] pai = new int[target + 1];

        for (int i = 1; i <= target; i++) {
            f[i] = 0;
            for (int j = 0; j < nums.length; j++) {
                if(i >= nums[j]) {
                    f[i] += f[i - nums[j]];
                    if(f[i - nums[j]] > 0) {
                        pai[i] = nums[j];
                    }
                }
            }
        }

        // 输出方案，pai 数组存放达到该 target 的某个方案
        if(f[target] > 0) {
            int i = target;
            while(i != 0) {
                System.out.print(pai[i] + " ");
                i -= pai[i];
            }
        } 

        return f[target];
    }
```





### 3、最值型背包问题



#### 1.背包问题 II 125

有 `n` 个物品和一个大小为 `m` 的背包. 给定数组 `A` 表示每个物品的大小和数组 `V` 表示每个物品的价值。问最多能装入背包的总价值是多大?

**样例 1:**

```
输入: m = 10, A = [2, 3, 5, 7], V = [1, 5, 2, 4]
输出: 9
解释: 装入 A[1] 和 A[3] 可以得到最大价值, V[1] + V[3] = 9 
```

**样例 2:**

```
输入: m = 10, A = [2, 3, 8], V = [2, 5, 8]
输出: 10
解释: 装入 A[0] 和 A[2] 可以得到最大价值, V[0] + V[2] = 10
```

**挑战**

O(nm) 空间复杂度可以通过, 不过你可以尝试 O(m) 空间复杂度吗?

**注意事项**

1. `A[i], V[i], n, m` 均为整数
2. 你不能将物品进行切分
3. 你所挑选的要装入背包的物品的总大小不能超过 `m`
4. 每个物品只能取一次

 【分析】

由下图分析可知，最大重量不一定为最大价值。

![image-20200905085713602](assets/blog_image/2020-09-02-DP-include2/image-20200905085713602.png)

**1、确定状态**：又分为 最后一步 和 化为子问题 

    1. **最后一步**：最后一个物品重量 （重量 An-1，价值 Vn-1）是否进入背包
       - 选择一：如果前N-1个物品能拼出W，最大总价值是V，前N个物品也能拼出W并且总价值是V。
       - 选择二：如果前N-1个物品能拼出W-AN，最大总价值是V，则再加上最后一个物品（重量An-1，价值Vn-1），能拼出W，总价值是V+Vn-1
       - ![image-20200905090151411](/assets/blog_image/2020-08-17-DP-Include/image-20200905090151411.png)
    2. **化为子问题：**
    
       - 原问题：要求前 N 个物品能不能拼出重量0，1，…，M，以及拼出重量W能获得的最大价值
       - 子问题：要求前 N-1 个物品能不能拼出重量0，1，…，M，以及拼出重量W能获得的最大价值
       - 状态：设 f[i] [w] = 用前i个物品拼出重量w时**最大总价值**（-1表示不能拼出w）

**2、转移方程**

![image-20200905090715137](/assets/blog_image/2020-09-02-DP-include2/image-20200905090715137.png)

**3、初始条件和边界情况**

- 初始条件：
  f[0] [0]=0：0个物品可以拼出重量0，最大总价值是0
  f[0] [1..M] = -1 表示 0 个物品不能拼出大于0的重量
  边界情况：
  -f[i-1] [w-Ai]只能在w>=Ai，并且f[i-1] [w-Ai-1] != -1时使用

**4、计算顺序**

按行按列来，最终结果为最后  1 列的 max 值。 

```java
 // 二维数组
    public int backPackII(int m, int[] A, int[] V) {
        int n = A.length;
        if(n == 0) {
            return 0;
        }

        int[][] f = new int[n+1][m + 1];
        f[0][0] = 0;//表示 0 个物品
        for (int i = 1; i < f.length; i++) {
            f[0][i] = -1;
        }

        for (int i = 1; i < n+1; i++) {
            for (int j = 0; j < m+1; j++) {
                f[i][j] = f[i-1][j];
                if(j>=A[i-1] && f[i-1][j-A[i-1]] != -1) {
                    f[i][j] = Math.max(f[i][j], f[i-1][j-A[i-1]] + V[i-1]);
                }
            }
        }

        int res = Integer.MIN_VALUE;
        for (int i = 0; i < m+1; i++) {
            res = Math.max(res, f[n][i]);
        }
        return res;
    }
/*------------------------------------------------------------*/
    // 一维数组
    public int backPackII_2(int m, int[] A, int[] V) {
        int n = A.length;
        if(n == 0) {
            return 0;
        }

        int[] f = new int[m + 1];
        f[0] = 0;
        for (int i = 1; i <= m; i++) {
            f[i] = -1;
        }
        
        for (int i = 1; i <= n; i++) {
            for (int j = m; j >= 0; j--) {
                // 还未被置换的f[j]存放的是 f[i-1][j]
                // 置换后的存放的是 f[i][j]
                if(j >= A[i-1] && f[j - A[i-1]] != -1) {
                    f[j] = Math.max(f[j], f[j - A[i-1]] + V[i-1]);
                }
            }
        }

        int res = Integer.MIN_VALUE;
        for (int i = 0; i < m+1; i++) {
            res = Math.max(res, f[i]);
        }
        return res;
    }
```

### 1、硬币排成线 394

有 `n` 个硬币排成一条线。两个参赛者轮流从右边依次拿走 1 或 2 个硬币，直到没有硬币为止。拿到最后一枚硬币的人获胜。

请判定 **先手玩家** 必胜还是必败?

**【解答】**

**1、动态规划组成部分一：确定状态**
博弈动态规划**通常从第一步分析**，而不是最后一步——因为局面越来越简单，石子数越来越少

- 面对 N 个石子，先手 Alice 第一步可以拿 1 个或 2 个石子
- 这样后手Bob就面对N-1个石子或N-2个石子
- 先手Alice一定会选择能让自己赢的一步
- 因为双方都是采取最优策略



必胜：取走一个或者两个石子，可以使对手必败

必败：无论取走什么，对方都是必胜

![image-20200919100912339](/assets/blog_image/2020-09-02-DP-include2/image-20200919100912339.png)

**子问题**

- 要求面对N个石子，是否先手必胜
- 需要知道面对N-1个石子和N-2个石子，是否先手必胜
- 子问题
- 状态：设表示面对i个石子，是否先手必胜（ f[i]=TRUE/FALSE）

**2、转移方程**

对于自己而言，上一步称为对方的先手

![image-20200919101327043](/assets/blog_image/2020-09-02-DP-include2/image-20200919101327043.png)

**3、初始条件和边界情况**

- 设 f[N] 表示面对 N 个石子，是否先手必胜（ f[N]=TRUE/FALSE ）
- f[i] = f[i-1] == FALSE    OR     f [i-2] == FALSE
- f[0] = FALSE —— 面对0个石子，先手必败
- f[1] = f[2] = TRUE —— 面对1个石子或2个石子，先手必胜

**4、计算顺序**

f[0]，f[1]，[2]……f[N]
如果 f[N] = TRUE 则先手必胜，否则先手必败。

时间复杂度O（N）空间复杂度O（N），可以滚动数组优估至O（1）





### 2、硬币排成线 III 396

【题意】
给定一个序列a[0]，a[1]，....，a[N-1]，两个玩家Alice和Bob轮流取数，每个人每次只能取第一个数或最后一个数双方都用最优策略，使得自己的数字和尽量比对手大。

问先手是否必胜（如果数字和一样，也算先手胜)
【例子】
输入：[1，5，233，7]
输出：True（先手取走1，无论后手取哪个，先手都能取走233）

【解析】

博弈题的思路是找到先手和后手的状态相同点。

​	此题采用价格差作为状态。设己方数字和是A，对手数字和是B，即目标是 A>=B 等价于 A-B>=0 。也就是说，如果 Alice 和 Bob 都存着自己的数字和与对手的数字和之差，分别记为 SA=A-B，SB=B-A 。则 Alice 的目标是最大化 SA ， Bob 的目标是最大化 SB。

- 当一方X面对剩下的数字，可以认为X就是当前的先手，他的目标就是最大化SX=X-Y
- 当他取走一个数字m后，对手Y变成先手，同理他也要最大化SY=Y-X
- 对于X来说，**Sx = - SY + m** （Sx = X- Y,Sy = Y - (X+m), 所以 Sx = -Sy + m)
- 其中，m是当前这步的数字，-SY，是对手看来的数字差取相反数（因为先手是X）
- 现在X有两种选择，取第一个数字m1或最后一个数字m2，为了最大化Sx，应该选择较大的那个Sx

1、确定状态：

1. 如果Alice第一步取走a[0]，Bob面对a[1..N-1]
   Bob的最大数字差是Sy ，Alice的数字差是a[0]-Sy，
2. 如果Alice第一步取走a[N-1]，Bob面对a[0..N-2]
   Bob的最大数字差是S'y Alice的数字差是a[N-1]-S'y
3. 然后Alice 需要在两个数字差中，选择最大的。

子问题：

当Bob面对a[1..N-1]，他这时是先手。他的目标同样是最大化先手（自己）和后手（Alice）的数字差。但是此时的数字少了一个：a[1..N-1]。构成子问题。

因此状态为：

设 f[i] [j] 为**一方先手**在面对 a[i...j] 这些数字时，能得到的最大的与对手的数字差。

2、转移方程

![image-20200923104633680](/assets/blog_image/2020-09-02-DP-include2/image-20200923104633680.png)

​	值得思考的是，f(i, j) 之所以求值为f(i+1,j)的负数，是由于f(i+1,j) 是先手为对方时，所求得的最大值。很明显，该问题也是区间型DP。

3、初始条件与边界情况

由于是区间型DP，因此初始化从区间长度为1 开始。即：

只有一个数字 a[i] 时，己方得 a[i] 分，对手0分，数字差为 a[i]
	f [i] [i] = a[i]（i=0，1.…，N-1）

4、计算顺序

​	从区间长度为 1，到区间长度为 N。最后判断为：如果 f[0] [N-1] >= 0(数字和一样，也算先手赢)。

时间复杂度：O(N^2)， 空间复杂度 O(N^2)

### 3、攀爬字符串 430

```txt
      great
     /    \
    gr    eat
   / \    /  \
  g   r  e   at
             / \
            a   t
```

在攀爬字符串的过程中, 我们可以选择其中任意一个非叶节点, 交换该节点的两个子节点.

例如，我们选择了 `"gr"` 节点, 并将该节点的两个子节点进行交换, 并且将祖先节点对应的子串部分也交换, 最终得到了 `"rgeat"`. 我们认为 `"rgeat"` 是 `"great"` 的一个攀爬字符串.

```txt
      rgeat
     /    \
    rg    eat
   / \    /  \
  r   g  e   at
             / \
            a   t
```

类似地, 如果我们继续将其节点 `"eat"` 和 `"at"` 的子节点交换, 又可以得到 `"great"` 的一个攀爬字符串 `"rgtae"`.

```txt
     rgtae
     /    \
    rg    tae
   / \    /  \
  r   g  ta   e
         / \
        t   a
```

```
解释: 如同上面描述的.
```

**样例 2:**

### 4、扎气球   168

【题目】

​	有n个气球，编号为`0`到`n-1`，每个气球都有一个分数，存在`nums`数组中。每次吹气球i可以得到的分数为 `nums[left] * nums[i] * nums[right]`，left和right分别表示`i`气球相邻的两个气球。当i气球被吹爆后，其左右两气球即为相邻。要求吹爆所有气球，得到最多的分数。

**样例 1:**

```

输出：270
解释：
nums = [4, 1, 5, 10] 吹爆 1, 得分 4 * 1 * 5 = 20
nums = [4, 5, 10]   吹爆 5, 得分 4 * 5 * 10 = 200 
nums = [4, 10]   吹爆 4, 得分 1 * 4 * 10 = 40
nums = [10]   吹爆 10, 得分 1 * 10 * 1 = 10

```

**样例 2:**

```
nums = [3, 1, 5] 吹爆 1, 得分 3 * 1 * 5 = 15
nums = [3, 5] 吹爆 3, 得分 1 * 3 * 5 = 15
nums = [5] 吹爆 5, 得分 1 * 5 * 1 = 5
```

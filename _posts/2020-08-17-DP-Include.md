### 1、换硬币（最值DP）

​	给出不同面额的硬币以及一个总金额. 写一个方法来计算给出的总金额可以换取的最少的硬币数量. 如果已有硬币的任意组合均无法与总金额面额相等, 那么返回 `-1`.

```
你可以假设每种硬币均有无数个
 总金额不会超过10000
 硬币的种类数不会超过500, 每种硬币的面额不会超过100
```

**样例1**

```
输入：
[1, 2, 5]
11
输出： 3
解释： 11 = 5 + 5 + 1
```

【解答】

按照动态规划的思考步骤：

**1、确定状态**：又分为 最后一步 和 化为子问题 

 1. 最后一步：虽然我们不知道最优策略是什么，但是最优策略肯定是K枚硬币a1，a2…，ak面值加起来是总金额 S，即最后一步就是 k-1枚硬币已经得到 S-aK的面值，且由于是最优策略，前面27-aK的硬币数一定是最少的。

    所以我们就要求：最少用多少枚硬币可以拼出 S-ak

    原问题是最少用多少枚硬币拼出 S

 2. 我们将原问题转化成了一个子问题，而且规模更小：S-ak 为了简化定义，我们设状态f(X) = 最少用多少枚硬币拼出X

**2、转移方程**: 现假设，硬币面值为 2，5，7。那么

![image-20200818081036562](/assets/blog_image/2020-08-17-DP-Include/image-20200818081036562.png)

​	因此转移方程便为：

> [X]=min(f[X-2]+1,f[X-5]+1,f[X-7]+1}

**3、初始条件和边界情况**

初始条件：f[0] = 0 表示可以到达 0 元，且为 0 个硬币

边界：

1. 刚开始计算时，之后还未到达的 f[n] 置为 MAX_VALUE，表示拼不出来 
2. 对于 X-2，X-5，X-7 可能小于 0 而言，加上一个判断语句，只能当前钱数大于硬币面值数才可以进行转移方程的计算。

**4、计算顺序**

一般都是从前往后计算，因为之后的结果依靠于之前结果。

```
        f[0] = 0; // init 
        for (int i = 1; i <= amount; i++) {
            f[i] = Integer.MAX_VALUE; // 初始化，表示不可到达
            for (int j = 0; j < n; j++) {

                    }
            }
        }
```



### 2、青蛙的跳跃游戏（存在性DP）

​	给出一个非负整数数组，你最初定位在数组的第一个位置。数组中的每个元素代表你在那个位置可以跳跃的最大长度。判断你是否能到达数组的最后一个位置。

> 数组`A`的长度不超过5000，每个元素的大小不超过5000

***样例 1***

```
输入 : [2,3,1,1,4]
输出 : true
在0上跳1（<=2）,在1上跳3（<=3）
```

【解答】

**1、确定状态**：又分为 最后一步 和 化为子问题 

 1. 最后一步：

    ![image-20200818084608501](/assets/blog_image/2020-08-17-DP-Include/image-20200818084608501.png)

    · 那么，我们需要知道青蛙能不能跳到石头 i（i<n-1）
    · 而我们原来要求青蛙能不能跳到石头 n-1

 2. 我们将原问题转化成了一个子问题，而且规模更小：设 f[j] 表示青蛙能不能跳到石头 j

**2、转移方程**: 

​	将上一个石头进行枚举，找出能跳到上一个，且也能跳到下一个的石子，bool数组进行标记

![image-20200818084840935](/assets/blog_image/2020-08-17-DP-Include/image-20200818084840935.png)

**3、初始条件和边界情况**

初始条件：f[0]=True，因为青蛙一开始就在石头0

边界： 并没有边界情况，由前至后。

**4、计算顺序**

一般都是从前往后计算，因为之后的结果依靠于之前结果。

```
public boolean canJump(int[] A) {
        if(A == null || A.length == 0){
            return false;
        }

        int n = A.length;
        boolean[] f = new boolean[n+1];
        // 1.初始化
        f[0] = true;
        for (int i = 1; i <= n; i++) {
            f[i] = false;
            for (int j = 0; j < i; j++) { // 遍历之前的
                if(f[j] && A[j] + j >= i){
                    f[i] = true;
                    break;
                }
            }
        }
        return f[n-1];
    }
```

### 1、最长连续子序列

![image-20200821092758475](/assets/blog_image/2020-08-17-DP-Include/image-20200821092758475.png)

**1、确定状态**：又分为 最后一步 和 化为子问题 

  1. **最后一步**：对于最优的策略，一定有最后一个元素 a[i]
     	第一种情况：最优策略中最长连续上升子序列就是 {aj} ，答案是 1
          	第二种情况，子序列长度大于1，那么最优策略中 aj[j] 前一个元素肯定是 a[j-1]。这种情况一定是 a[j-1] < a[j]
          	 因为是最优策略，那么它选中的以 a[j-1] 结尾的连续上升子序列一定是最长的
  2. **化为子问题：**本来是求以a[i]]结尾的最长连续上升子序列，现要求以a[j-1]结尾的最长连续上升子序列 ----->>化为子问题
            状态：设 f[j] = 以 a[j] 结尾的最长连续上升子序列的长度

**2、转移方程**

![image-20200821093732335](/assets/blog_image/2020-08-17-DP-Include/image-20200821093732335.png)

**3、初始状态：**无，开始先置为 1 ，之后再判断是否大于上一个，要是大于就相加。

**4、计算顺序：**值得注意的是，最终结果并不一定是 f[n-1] 。

且由于题目为左右都要判断，因此还需要将序列转置，以做到两边都判断。

关于空间优化。之前采用的是与序列等长数组，但是由转移方程可以看出，当前的大小只需要前一个的值。因此可以优化为滚动数组。用 now 值表示当前值， old 表示上一个值。

```
	public int longestIncreasingContinuousSubsequence(int[] A) {
        int n = A.length;
        if(n == 0){
            return 0;
        }
        // 函数计算最长子序列
        // 之所以安置为函数，是因为之后还需要反转进行再判断
        calc(A, n);
        A = reverse(A, n);
        calc(A, n);

        return result;
     }
    
    void calc(int[] A, int n) {
        int now = 0, old = 1;// 开始赋为0 和 1 是为了与后面的循环达成一致
        int[] f = new int[2];
        for (int i = 0; i < n; i++) {
            old = now;
            now = 1- old;

            f[now] = 1;
            if(i > 0 && A[i-1] < A[i]){
                f[now] = f[old] + 1;
            }
            if(f[now] > result){
                result = f[now];
            }
        }
    }
```

### 2、网格最短路径和（滚动数组优化）

![image-20200821092731927](/assets/blog_image/2020-08-17-DP-Include/image-20200821092731927.png)

**1、确定状态**：又分为 最后一步 和 化为子问题 

 1. **最后一步**：到达最后一步的路径 = 倒数第二步的路径

    又因为，到达最后一步可能来自上和左两个方向，因此 需要求出到达 左 和 上 两个点（如果存在的话）的路径，再将两条路径判断出最小的之后，赋予到当前点上。 

 2. **化为子问题：**本来是求以 a[ i ] [ j ] 结尾的数值和，现要求以 a[ i-1] [j] 和 a[i] [j-1] 结尾的数值和 ----->>化为子问题
           状态：设 f[i] [j] =  到达 [i] [j] 点的数值和

**2、转移方程**

> f [i] [j] = min { f[i-1] [j]] ， f[i] [j-1] } + grid[i] [j]

**3、初始条件和边界情况**
	初始条件：f[0] [0] = grid[i] [j]，因为机器人只有一种方式到左上角
	边界情况：i=0或j=0，则前一步只能有一个方向过来→f[i] [j] = 1 (第一行或者第一列)

**4、计算顺序**

> f[0] [0]=0
> 计算第0行：f[0] [0]，f[0] [1]..…，f[0] [n-1]，
> 计算第1行：f[1] [0]，f[1] [1].…，f[1] [n-1]
> 计算第m-1行：f[m-1] [0]，f[m-1] [1]..…，f[m-1] [n-1]
> 答案是f[m-1] [n-1]

**5、空间优化**

​	现对滚动数组进行讨论：之前方法，开一个 m*n 的二维数组，实际上，有转移方程可以看出，每一个值的计算，只需要上左两个点的值，因此在算每一行的值时，存储上一行即可。

​	实际操作时，可以不用每次开数组，而是用滚动法计算f[0] [0]...f[0] [n-1]，计算 f[1] [0]...f[1] [n-1]
计算 f[2] [0..n-1]时，把值写在f[0] [0..n-1]的数组里。

​	同理，f[3] [0.n-1]写在 f[1] [0..n-1]的数组里....

​	最后f[m-1] [n-1]存储在 f[0] [n-1]（或者f[1] [n-1]）里，直接输出。

```
	public int minPathSum(int[][] grid) {
		int m = grid.length;
		int n = grid[0].length;

		// 空间优化
		int[][] f = new int[2][n];
		int old = 1, now = 0;
		// 初始化
		f[0][0] = 0;

		int up, left;
		for (int i = 0; i < m; i++) {
			// 不断进行反转 ———— 0行 和 1行 的翻转
			old = now; // old 为 i-1 行 
			now = 1 - now; // now is row i 
			for (int j = 0; j < n; j++) {
				if(i== 0 && j==0){
					f[now][j] = grid[i][j];
					continue;
				}
				f[now][j] = grid[i][j];
				if(i > 0) {
					up = f[old][j];
				}else {
					up = Integer.MAX_VALUE;// 表示还未到达
				}

				if(j > 0){
					left = f[now][j-1];
				}else {
					left = Integer.MAX_VALUE;
				}
				// 取小值进行赋予
				if(up < left){
					f[now][j] += up;
				}else {
					f[now][j] += left;
				}
			}
		}
				return f[now][n-1];
	}
```



### 3、炸弹 killed 最大敌人数

每一个格子可能是一堵墙 W,或者 一个敌人 E 或者空 0 (数字 '0'),*

![image-20200821101510772](/assets/blog_image/2020-08-17-DP-Include/image-20200821101510772.png)

​	把炸弹放在 (1,1) 能杀3个敌人

【解答】

 1. 首先，由于炸弹可以向四个方向爆炸，那么分析一个方向即可（比如分析向上方向）。若采用直接枚举，向上枚举到碰到墙为止，时间复杂度为 O(n* M *M)。

 2. 现**采用 DP 思想**，首先规定一个前提——我们假设有敌人或有墙的格子也能放炸弹

    - 有敌人的格子：格子里的敌人被炸死，并继续向上爆炸
    - 有墙的格子：炸弹不能炸死任何敌人

    在（i，j）格放一个炸弹，它向上能炸死的敌人数是：
    -（i，j）格为空地：（i-1，j）格向上能炸死的敌人数
    -（i，j）格为敌人：（i-1，j）格向上能炸死的敌人数+1
    -（i，j）格为墙：0

    因此：**子问题**为——
    ●需要知道(i-1, j)格放一个炸弹向上能炸死的敌人数
    ●原来要求(i, j)格放一个炸弹向上能炸死的敌人数
    ●因此转换为子问题
    状态:	Up[i] 表示(i, j)格放一个炸弹向上能炸死的敌人数

	3. 为什么在有敌人的地方规定也能放炸弹？这是为了空格处做基础，最终只需要提取出空格处敌人数即可

	4. **初始条件**：第0行的Up值和格子内容相关(由于是从左向右，从上至下，因此初始化第一行即可)
    -Up[0] [j] = 0，如果（0，j）格不是敌人
    -Up[0] [j] = 1，如果（0，j）格是敌人

	5. 之后便按照相同方式，计算四个方向即可。最终对四个方向加起来，取出空格处炸死敌人最多处

```
		// up 方向
		for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if(grid[i][j] == 'W'){// 遇到墙
                    f[i][j] = 0;
                }else {
                    f[i][j] = 0;

                        f[i][j] = 1; 
                    }

                    if(i > 0) {
                        f[i][j] += f[i-1][j];
                    }

                }
                res[i][j] += f[i][j]; 
            }
        }
```



### 4、数 1

给出一个 **非负** 整数 num，对所有满足 `0 ≤ i ≤ num` 条件的数字 i 均需要计算其二进制表示中数字 1 的个数并以数组的形式返回。

```
输入： 5
输出： [0,1,1,2,1,2]
解释：
0~5的二进制表示分别是：
000
001
010
011
100
101
每个数字中1的个数为： 0,1,1,2,1,2
```

【解答】

法一：直接对每一位对 2 取模进行判断。

![image-20200822100405587](/assets/blog_image/2020-08-17-DP-Include/image-20200822100405587.png)

法二：采用DP思想：

 1. 最后一步：

    > eg: 71:01000111
    >
    > ​	  35:0100011
    >
    > ​	因此 求 71 二进制的 1 的个数，等于35 中1 的个数加上最后一位是否为 1

    因此子状态为：

    - 要求N的二进制表示中有多少1
    - 在N的二进制去掉最后一位后 mod 2，设新的数是Y=（X>>1）（右移一位）
    - 需要知道 Y 的二进制表示中有多少个 1
    - 子问题
      状态：设 f[i]] 表示 i 的二进制表示中有多少个1

	2. 状态转移方程

    > f[i]] = f[i>>1] + (i mod 2)

	3. 初始：f[0] = 0，顺序由小到大，时间空间复杂度都为 O(1)

### 1、涂房子I（三个颜色）

【题目】

这里有`n`个房子在一列直线上，现在我们需要给房屋染色，分别有**红色蓝色和绿色**。每个房屋染不同的颜色费用也不同，你需要设计一种染色方案使得**相邻的房屋颜色不同**，并且费用最小，返回最小的费用。

费用通过一个`n`x`3` 的矩阵给出，比如`cost[0][0]`表示房屋`0`染红色的费用，`cost[1][2]`表示房屋`1`染绿色的费用。

### 样例

```
输入: [[14,2,11],[11,14,5],[14,3,10]]
输出: 10
解释: 第一个屋子染蓝色，第二个染绿色，第三个染蓝色，最小花费：2 + 5 + 3 = 10.
```

【解析】

1.DP思想套路

- 最后一步：最优策略中房子N-1一定染成了红、蓝、绿中的一种

- 但是相邻两栋房子不能漆成一种颜色

  - 所以如果最优策略中房子N-1是红色，房子N-2只能是蓝色或绿色
  - 所以如果最优策略中房子N-1是蓝色，房子N-2只能是红色或绿色
  - 所以如果最优策略中房子N-1是绿色，房子N-2只能是红色或蓝色

  过于复杂....

![image-20200824120409652](/assets/blog_image/2020-08-17-DP-Include/image-20200824120409652.png)

**序列+状态**：若像之前一样，只记录一维DP，复杂，因此需要记录 三个颜色房子的分别cost。

因此 f[i] [j] 表示前 i 栋房子，且第 i-1 栋为 j 色的最小花费。此处 i-1 栋，代表的是以 0 开头的数组下标。 

![image-20200826082558719](/assets/blog_image/2020-08-17-DP-Include/image-20200826082558719.png)

```
f[i][0] = min{f[i-1][1]+cost[i-1][0],f[i-1][2]+cost[i-1][0]}
f[i][1] = min(f[i-1][0]+cost[i-1][1],fli-1][2]+cost[i-1][1]}
f[i][2] = min(f[i-1][0]+cost[i-1][2],fi-1][1]+cost[i-1][2l}
```

所以最终答案最小值是，min（f[N] [0]，f[] [1]，f[N] [2]}。时间复杂度O（N），空间复杂度O（N）

### 2、涂房子II

【题目】	与上一题大致一致，区别在于现在颜色为 k 种

​	这里有`n`个房子在一列直线上，现在我们需要给房屋染色，共有`k`种颜色。每个房屋染不同的颜色费用也不同，你需要设计一种染色方案使得相邻的房屋颜色不同，并且费用最小。

​	费用通过一个`n`x`k` 的矩阵给出，比如`cost[0][0]`表示房屋`0`染颜色`0`的费用，`cost[1][2]`表示房屋`1`染颜色`2`的费用。

​	样例

```plain
输入:
costs = [[14,2,11],[11,14,5],[14,3,10]]
输出: 10
说明:
三个屋子分别使用第1,2,1种颜色，总花费是10。
```

因此沿用之前 3 个颜色时的想法。

#### 现在进行一个优化小技巧

​	我们发现：对于除了此时除掉的是最小值外，那么当前整个序列的最小值都不变。因此，用两个临时变量记录下次小值 以及 最小值即可。

![image-20200826084022137](/assets/blog_image/2020-08-17-DP-Include/image-20200826084022137.png)

- 记录下f[i-1] [1] …，f[i-1] [K] 中的最小值和次小值分别是哪个
- 假如最小值是 f[i-1] [a]，次小值是 f[i-1] [b]
- 则对于 j=1，2，3.，a-1，a+1...，K，f[i] [j] = f[i-1] [a] + cost [i-1] [j] 。即前 i-1 栋的最小值都不变
- f[i] [a] = f[i-1] [b] + cost[i-1] [a].
- 时间复杂度降为O（NK）

阶段也可以变化：买或卖

- 在阶段2，**卖了一股后当天买一股**，进入阶段4。

- 阶段3：买卖过一次

要求f[N] [1]，…，f[N] [5]
需要知道f[N-1] [1]…，f[N-1] [5]
子问题
状态：f[i] [j]表示前天（第i-1天）结束后，在阶段的最大获利

#### 4.买卖股票IV（DP）

【题目】

给定数组 `prices`, 其中第 `i` 个元素代表某只股票在第 `i` 天第价格.

**你最多可以完成 `k` 笔交易.** 问最大的利润是多少?

**样例 1:**

```
输入: k = 2, prices = [4, 4, 6, 1, 1, 4, 2 ,5]
输出: 6
解释: 以 4 买入, 以 6 卖出. 然后再以 1 买入, 以 5 卖出. 利润为 2 + 4 = 6.
```

【题解】

首先，如果K很大，K>N/2, 则题目可以化简成为**买卖股票II**，相当于买卖任意多次。

其次，可以借鉴之前III的思路，分为 2K+1 个阶段

![image-20200827111452829](/assets/blog_image/2020-08-17-DP-Include/image-20200827111452829.png)

![image-20200827111459028](/assets/blog_image/2020-08-17-DP-Include/image-20200827111459028.png)

```
        for (int i = 1; i <= n; i++) {

            // 对 1 、 3、 5 无股票阶段
            for (int j = 1; j <= 2 * K + 1; j += 2) {
                f[i][j] = f[i - 1][j];
                if (i > 1 && j > 1 && f[i - 1][j - 1] != Integer.MIN_VALUE) {
                    f[i][j] = Math.max(f[i][j], f[i - 1][j - 1] + prices[i - 1] - prices[i - 2]);
                }
            }

            // 对 2 、 4 持有股票阶段
            // 需要考虑 昨天的股票 今天卖了 又买了新的
            for (int j = 2; j < 2 * K + 1; j += 2) {
                f[i][j] = f[i - 1][j - 1];
                if (i > 1 && f[i - 1][j] != Integer.MIN_VALUE) {
                    f[i][j] = Math.max(f[i][j], f[i - 1][j] + prices[i - 1] - prices[i - 2]);
                }

                if (j > 2 && i > 1 && f[i - 1][j - 2] != Integer.MIN_VALUE) {
                    f[i][j] = Math.max(f[i][j], f[i - 1][j - 2] + prices[i - 1] - prices[i - 2]);
                }
            }
        }
```

因此时间复杂度为 O(NK)，空间复杂度为 O(NK)，可以优化为O(K)

### 2、分割回文串 II 108

最少需要分割几次?

**样例 1:**

**样例 2:**

    1. **最后一步**：最后为 s 的前 n-j 个字符，和一个长度为 j 的回文串

       - 状态：设S前 i 个字符 S[0...i-1] 最少可以划分成 f[i] 个回文串

**2、转移方程**：对 j 进行枚举

![image-20200831091904545](/assets/blog_image/2020-08-17-DP-Include/image-20200831091904545.png)

**3、初始情况：**空串可以被分成0个回文串		f[0]=0

**4、现说明**最重要的：如何找到回文串。

- 从 S 每一个字符开始向两边扩展

  考虑奇数长度回文串和偶数长度回文串

- 用 isPalin[i] [j] 表示 S[i.j] 是否是回文串

- 时间复杂度O（N^2）

```
        for (int i = 0; i < n; i++) {
            int p = i, q = i;
            // 先判断奇数长度的回文串
            while (p >= 0 && q < n && sc[p] == sc[q]) {
                isPlain[p][q] = true;
                q++;
                p--;
            }

            // 判断偶数长度的回文串
            p = i;
            q = i + 1;
            while (p >= 0 && q < n && sc[p] == sc[q]) {
                isPlain[p][q] = true;
                q++;
                p--;
            }
        }
```

**5、值得注意的是**，由于 f[n] 表示前n项序列的最少可以划分为 n 个回文串，因此最终应将 f[n] -1 ，因为所求为最小划分。




### 3、书籍复印 437

【题目】

给定 `n` 本书, 第 `i` 本书的页数为 `pages[i]`. 现在有 `k` 个人来复印这些书籍, 而每个人只能复印编号连续的一段的书, 比如一个人可以复印 `pages[0], pages[1], pages[2]`, 但是不可以只复印 `pages[0], pages[2], pages[3]` 而不复印 `pages[1]`.

所有人复印的速度是一样的, 复印一页需要花费一分钟, 并且所有人同时开始复印. 怎样分配这 `k` 个人的任务, 使得这 `n` 本书能够被尽快复印完?

返回完成复印任务最少需要的分钟数.

**样例 1:**

```
输入: pages = [3, 2, 4], k = 2
输出: 5
解释: 第一个人复印前两本书, 耗时 5 分钟. 第二个人复印第三本书, 耗时 4 分钟.
```

**样例 2:**

```
输入: pages = [3, 2, 4], k = 3
输出: 4
解释: 三个人各复印一本书.
```

**挑战**：时间复杂度 O(nk)

**注意事项**：书籍页数总和小于等于2147483647

**【解答】**

首先进行题目分析：如果一个抄写员抄写第 i 本到第 j 本书，则需要时间 A[i]+A[i+1]+...+A[j]
最后完成时间取决于耗时最长的那个抄写员。需要找到一种分段方式，分成**不超过K段**，使得所有段的数字之和的最大值最小。

**1、确定状态**：又分为 最后一步 和 化为子问题 

  1. **最后一步**：最优策略中最后一个抄写员Bob（设他是第K个）抄写的部分

     - 一段连续的书，包含最后一本
     - 如果Bob抄写第 j 本到第 N-1 本书
     - 则Bob需要时间 A[j]+...+A[N-1]
     - 需要知道前面 K-1 个人最少需要多少时间抄完前j本书（第 0~j-1 本书）

  2. **化为子问题：**求 K 个人最短需要多少时间抄完前 N 本书，需要知道 K-1 个人最少需要多少时间抄完前 j 本书。

     状态：设 f[k] [i] 为 k 个抄写员最少需要多少时间抄完前 i 本书。

**2、转移方程**：对 j 进行枚举

![image-20200831101451326](/assets/blog_image/2020-08-17-DP-Include/image-20200831101451326.png)

j = 0 是由于可以安排第一个抄写员不抄

j = i 是由于可以安排最后一个抄写员不抄

**3、初始条件**

- 0个抄写员只能抄0本书
- f[0] [0]=0，f[0] [1]=f[0] [2]=....=f[0] [N]=正无穷（表示不能抄
- k个抄写员（k>0）需要0时间抄0本书
- f[k] [0]=0（k>0）

**4、计算顺序**

- 计算f[0] [0]，f[0] [1]，f[0] [N]
- 计算f，f[1] [1]…，f[1] [N]
- 计算f[K] [0]，f[K] [1]..，f[K] [N]
- 答案是f[K] [N]
- 时间复杂度O（N2K），空间复杂度O（NK），优化后可以达到O（N）(滚动数组)

**5、一个小技巧**

由于遍历是要列举第 i 个人抄写共 j 本书时的累加和，明显会重复计算。因此可以从后往前（即从只抄写 0 本，一直到抄写 n 本）进行逆向累加。

```
        for (int i = 1; i <= k; i++) {
            // i 个人 0 本书 耗费 0分钟
            f[i][0] = 0;
            // 从第 1 本 ~ 第 n 本
            for (int j = 1; j <= n; j++) {// j 本书
                f[i][j] = Integer.MAX_VALUE;    // i个人 抄写 j本书
                sum = 0;
                for (int j2 = j; j2 >= 0; j2--) {
                    f[i][j] = Math.min(f[i][j], Math.max(f[i-1][j], sum));
                    if(j2 > 0) {// 逆向遍历
                        //这样就能让原本需要遍历计算 第 k个人需要复印书
                        // 变为累加和 sum
                        // 因为最后一本是 0
                        sum += pages[j2-1];
                    }
                }
            }
        }
```
